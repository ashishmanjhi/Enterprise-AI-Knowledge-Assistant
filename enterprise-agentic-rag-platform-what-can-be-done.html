1 Document Management

Upload PDF & DOCX
POST /api/v1/documents/upload. Files saved to data/raw/. Background task runs ingestion immediately. Max 10 MB. Supports only .pdf and .docx.
List & Search Documents
GET /api/v1/documents with DB-backed listing. Filename search filter in the UI. Pagination via skip/limit.
Delete Document
DELETE /api/v1/documents/{id} removes the file, soft-deletes all FAISS vectors, removes from BM25, then runs index compaction automatically.
Document Statistics
GET /api/v1/documents/stats/overview returns total docs, chunk counts, vector store size, extraction backend info.
Table Extraction (PDF)
pdfplumber extracts tables per page → Markdown [TABLE] chunks. Each table row/column is preserved and fully searchable.
Chart & Image AI (PDF)
llava:7b via Ollama describes charts, graphs, and diagrams as [CHART] text chunks. Runs locally, no API key needed. Requires ollama pull llava:7b.
OCR Fallback
pytesseract activates on pages with no extractable text (scanned PDFs). Optional install — skipped gracefully when absent.
Re-index / Update Document
Partial No endpoint to re-process an already-uploaded file. You must delete and re-upload to pick up changes.

2 Chat & RAG

RAG Chat (POST /api/v1/chat)
Ask questions; system retrieves relevant chunks, builds context, and generates an answer. Returns sources with chunk IDs, filenames, scores, and page numbers.
Streaming Chat (SSE)
POST /api/v1/chat/stream — tokens streamed as Server-Sent Events. Streamlit UI renders them token-by-token. Sources sent as a trailing sentinel line.
Direct LLM Chat (no RAG)
POST /api/v1/chat/direct — sends message straight to Ollama without any retrieval. Useful for general questions or testing the LLM baseline.
Query Reformulation
LLM rewrites vague or pronoun-heavy queries ("what does it say about…") using the last 3 conversation turns for context.
Query Expansion
Generates 3 alternative phrasings for the same question. Each expansion is retrieved independently; results fused via RRF.
HyDE Retrieval
Generates a short hypothetical answer and uses it as the FAISS embedding query. The original question is still used for BM25 — no keyword bleed.
Hybrid Retrieval (FAISS + BM25)
Both semantic and keyword search run in parallel via asyncio. Results merged with Reciprocal Rank Fusion. Configurable per-source weights.
Cross-Encoder Reranking
ms-marco-MiniLM-L-6-v2 scores each (query, chunk) pair and keeps only top-N. Enable per-request via use_reranking: true. Off by default.
Retrieval Method Selection
Per-request override: hybrid (default), faiss (semantic only), or bm25 (keyword only).
Source Attribution
Every response includes chunk IDs, filenames, page numbers, FAISS scores, BM25 scores, and ranks for full provenance.

Rate Limits (Phase 15)

POST /api/v1/chat → 20/min  |  POST /api/v1/chat/stream → 10/min  |  POST /api/v1/chat/direct → 30/min. Per IP. Returns HTTP 429 on breach. Disable with RATE_LIMIT_ENABLED=false.

3 Agentic RAG (LangGraph)

POST /api/v1/agent/chat — runs a full LangGraph state machine instead of the linear RAG chain. Limit: 10/min per IP.

NodeWhat it does
route_queryLLM classifies query → selects hybrid / faiss / bm25 strategy
rewrite_queryRewrites vague query before retrieval (configurable, max 2 rewrites)
retrieveRuns HybridRetriever with the chosen strategy
grade_documentsLLM scores each chunk YES/NO for relevance; irrelevant chunks dropped
generateBuilds context from graded chunks; calls LLM for final answer
check_groundingLLM verifies answer is supported by retrieved context (GROUNDED / HALLUCINATION)

If grounding fails and rewrite budget remains, the graph loops back and tries again with a different strategy. Response includes full trace (every node logged), grounding verdict, and rewrite count.

Speed on CPU-only Ollama

Each query runs 3–6 LLM calls. On CPU with qwen3:4b expect 2–5 minutes per query. Disable grading/grounding to speed up: AGENT_ENABLE_DOCUMENT_GRADING=false, AGENT_ENABLE_GROUNDING_CHECK=false.

4 Multi-Agent Ecosystem

POST /api/v1/multi-agent/chat — 5-agent pipeline. Limit: 5/min per IP.

Intent Classification
LLM classifies query as research, retrieval, or general. Research path triggers ResearchAgent; others go straight to RetrievalAgent.
Research Agent
Decomposes complex questions into ≤4 sub-questions. Answers each independently. LLM synthesises all findings into a research summary.
Retrieval Agent
Expands query into N variants, runs parallel multi-strategy retrieval for each, deduplicates by chunk_id, optionally reranks.
Knowledge Agent
Extracts named entities from the query, looks them up in the knowledge graph, builds enriched context string for generation.
Evaluation Agent
Scores the draft answer on source coverage, confidence, and length. Optional full RAGAS evaluation (EVAL_AGENT_USE_RAGAS=true).
Governance Agent
Runs full guardrails pipeline on the draft answer. Checks attribution completeness and confidence threshold. Can replace answer with a disclaimer.

Performance Reality

Research path runs 15+ LLM calls per query on CPU-only Ollama. Realistic time: 10–30 minutes for a complex research question. Best used with cloud LLM providers (OpenAI/Gemini) for practical speed.

5 Knowledge Graph

Build Knowledge Graph
POST /api/v1/kg/build — LLM extracts entities (PERSON, ORG, PRODUCT, LOCATION, CONCEPT, TECHNICAL) and relation triples from indexed documents. Regex fallback when LLM fails.
Query Entities & Subgraph
GET /api/v1/kg/query — find entities by name substring, expand to 2-hop ego-graph neighbourhood. Returns enriched context for RAG.
Browse Entities & Relations
GET /api/v1/kg/entities and /kg/relations — list all extracted entities and relation triples with confidence scores.
Hybrid Graph + Vector Retrieval
HybridGraphRetriever fuses FAISS+BM25 results with graph-based entity results using RRF (0.6 vector / 0.4 graph weights, configurable).
KG Statistics
GET /api/v1/kg/stats — node count, edge count, entity type distribution, top entities by degree.
Clear Knowledge Graph
DELETE /api/v1/kg/clear — wipes the graph and deletes the JSON persistence file.

6 Safety & Guardrails

All guardrails run on every chat request. Input checks run before the LLM; output checks run after generation.

DetectorMethodTriggersDefault Action
Prompt Injection14 regex patternsJailbreaks, instruction overrides, delimiter injection, role overrideBlock (HTTP 400)
PII Detection8 regex patternsSSN, credit cards, emails, phone numbers, API keys, DOB, IP addressesWarn + redact (no block)
Toxicity7 regex patternsViolence, self-harm, hate speech, harassmentBlock (HTTP 400)
HallucinationLLM-as-judge + token overlapAnswer uses facts not in retrieved contextWarn + replace answer

You can test guardrails directly via POST /api/v1/guardrails/check/input and /check/output, or check active settings via GET /api/v1/guardrails/status. All blocking behaviours are individually toggleable via env vars.

7 Conversational Memory

Per-Conversation History
Redis-backed message store, 24 h TTL. Automatically injected into every RAG prompt. Falls back to in-process dict if Redis is unavailable — transparent to the user.
History Summarisation
When a conversation exceeds the threshold (default 10 turns), the LLM summarises the oldest half into a single system message, keeping the context window small.
Memory API
GET /api/v1/memory/{id} — fetch full history. GET /api/v1/memory/{id}/info — metadata. DELETE /api/v1/memory/{id} — clear conversation.
Cross-Session Persistence
History survives Streamlit page refreshes. The chat page reloads history from Redis on startup via _restore_history_from_server().

8 Evaluation Framework (RAGAS)

MetricGround Truth Needed?Description
FaithfulnessNoAnswer is fully supported by retrieved context
Answer RelevancyNoAnswer actually addresses the question asked
Context PrecisionNoRetrieved chunks are relevant to the question
Context RecallYesContext contains all info needed to answer
Factual CorrectnessYesAnswer matches a reference/ground-truth answer

Trigger via POST /api/v1/evaluate or the 📊 Evaluate Streamlit page. Capped at 50 samples per run. Uses the configured Ollama model as LLM judge — no external API needed.

9 Auth & Security

JWT Bearer Auth
POST /auth/token — issue signed JWT. JWTAuthMiddleware enforces on all routes when AUTH_ENABLED=true. Transparent pass-through in development mode.
Role-Based Admin Guard
Admin endpoints require role=admin in the JWT claim. Verified by require_admin() FastAPI dependency. Separate admin/user passwords in .env.
Rate Limiting (Phase 15)
slowapi per-IP limits on all 5 LLM endpoints. X-Forwarded-For aware (respects Caddy reverse-proxy header). Returns 429 with Retry-After header.
JWT Secret Startup Check
Hard RuntimeError in production mode if secret is still the default. Warning in development. Prevents accidental insecure deployments.
CORS Configuration
Locked to localhost:8501 and localhost:3000 by default. Override with CORS_ORIGINS env var for production.
Password Security
Partial Constant-time comparison (hmac.compare_digest). Credentials loaded from .env. No bcrypt/argon2 hashing yet — passwords are plain strings.

10 Ops, Observability & Administration

Health Endpoints
GET /health (liveness), /healthz (Kubernetes), /readyz (readiness). All return JSON status with service name and version.
Service Status
GET /api/v1/status — checks Postgres, Redis, and Ollama connectivity in real time. Each service reported as connected / unreachable with latency.
Structured Logging
Rotating file handler (logs/app.log, 10 MB, 5 backups) + console. Every module uses get_logger(__name__) — no print() calls.
LangSmith Tracing
Every AgentGraph.run() and all its nodes traced in LangSmith when LANGSMITH_ENABLED=true + LANGSMITH_API_KEY set.
OpenTelemetry Spans
FastAPI HTTP spans + llm.generate / agent.run spans exported to any OTLP collector (Jaeger, Tempo, etc.) when OTEL_ENABLED=true.
PostgreSQL Schema
SQLAlchemy ORM + Alembic migration for documents, document_chunks, conversations, conversation_messages, feedback. Apply with alembic upgrade head.
User Feedback Collection
POST /api/v1/feedback — stores thumbs-up/down with conversation_id and response_id to data/feedback.jsonl. 👍👎 buttons in Chat UI.
Admin Endpoints
POST /api/v1/admin/clear-vector-stores — wipes FAISS + BM25 indices. GET /api/v1/admin/system-info — Python version, memory, disk, OS info.
Container Deploy (Podman)
deploy/podman/ has a full Compose stack: PostgreSQL, Redis, FastAPI, Streamlit, Caddy (TLS termination). Resource limits and health checks configured.

14 What's Still Missing / Incomplete

Agent Streaming (SSE)
Open Both agent endpoints return full JSON only. No token-by-token streaming for LangGraph execution — you wait for the complete response.
Feedback Analytics API
Open Thumbs data is written to JSONL but there is no endpoint to query it, aggregate scores, or export trends.
Agent & KG Unit Tests
Open The most complex code — AgentGraph, MultiAgentOrchestrator, GraphStore, EntityExtractor — has zero test coverage.
Document Re-indexing
Open No endpoint to re-process an existing document. Delete + re-upload is the only path.
Postgres Metadata Integration
Partial Schema exists and migration is applied. The ingestion pipeline is not yet fully wired to write all metadata fields to Postgres end-to-end.
Multi-Tenancy
Open All users share one FAISS + BM25 index. No namespace or tenant isolation. Not suitable for multi-user production without this.
Retry Logic (Ollama)
Open No automatic retry when Ollama times out on model load. A single cold-start failure returns a 500. Add tenacity retry for resilience.
Anthropic / Gemini Hot-Path
Open Provider implementations exist in backend/providers/ but are not wired to the RAG / agent hot path. Only Ollama and OpenAI are truly usable end-to-end.

Bottom Line

This is a fully functional, locally-running RAG platform for enterprise document Q&A. Upload your PDFs, ask questions in three different modes (linear RAG, agentic, multi-agent), get source citations, track conversation history, and evaluate quality — all without sending data to any external service. The main gaps for a real production deployment are: multi-tenancy, agent streaming, and complete Postgres wiring.
