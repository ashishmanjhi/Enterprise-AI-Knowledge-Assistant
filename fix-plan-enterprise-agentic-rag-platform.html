F-10 Move route-level module singletons to FastAPI Depends Effort: Medium
Root cause: rag_chain = RAGChain(), ingestion_pipeline = IngestionPipeline(), retriever = DocumentRetriever(), and _guardrails = GuardrailsPipeline() are instantiated at module import time. If any constructor throws (e.g. FAISS index missing), the entire app fails to start. They also cannot be mocked in tests.
Files: routes/documents.py, routes/chat.py, routes/agent.py

Fix (one file at a time): Replace module-level instantiation with a cached dependency function. The @lru_cache decorator gives you the singleton behaviour; Depends() gives you testability.

backend/api/routes/documents.py — example pattern (apply to all three files)
-# Initialize services -ingestion_pipeline = IngestionPipeline() -retriever = DocumentRetriever() +from functools import lru_cache +from fastapi import Depends + +@lru_cache(maxsize=1) +def get_ingestion_pipeline() -> IngestionPipeline: + return IngestionPipeline() + +@lru_cache(maxsize=1) +def get_retriever() -> DocumentRetriever: + return DocumentRetriever() # Update route signatures: -async def upload_document(background_tasks: BackgroundTasks, file: UploadFile = File(...)): +async def upload_document( + background_tasks: BackgroundTasks, + file: UploadFile = File(...), + pipeline: IngestionPipeline = Depends(get_ingestion_pipeline), +):

Sequence: Tackle documents.py first (most critical — it has two singletons). Then chat.py (RAGChain + GuardrailsPipeline). Then agent.py (already uses lazy _get_graph() pattern — keep as-is).
